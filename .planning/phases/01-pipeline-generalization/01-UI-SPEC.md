# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Pipeline Generalization.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | Streamlit native (built-in widgets only) |
| Icon library | none — Streamlit does not support icon libraries natively |
| Font | Streamlit default (Source Sans Pro via Streamlit theme) |

**Rationale:** The app is a pure Python/Streamlit application. No React, no Tailwind, no shadcn. All visual styling is Streamlit's native theme. The design contract is expressed as widget choices, layout structure, and copy — not CSS tokens.

---

## Spacing Scale

Streamlit controls all spacing. The contract below describes layout structure using Streamlit column ratios and widget stacking order, not pixel values.

| Concept | Streamlit Mechanism | Usage |
|---------|---------------------|-------|
| Column separation | `st.columns(2)` — equal split | Side-by-side upload widgets |
| Column separation | `st.columns([1, 5])` — narrow + wide | Back/Next button rows |
| Section break | `st.divider()` | Between major UI zones |
| Inline grouping | Widget stacking (no divider) | Dropdowns that belong to the same CSV |
| Vertical rhythm | One `st.divider()` between each logical zone | Upload zone, column-selection zone, proceed zone |

Exceptions: none. Do not introduce `st.empty()` spacers or raw HTML margin hacks.

---

## Typography

Streamlit controls all typography. The contract below specifies which heading level each element uses — do not upgrade or downgrade these.

| Element | Widget | Role |
|---------|--------|------|
| Page title | `st.title("Hydrology Reanalysis Dashboard")` | Consistent across all steps — do not change |
| Step label | `st.subheader("Step 1: Upload CSVs")` — source: existing pattern from Step 1 through Step 3 | Step identity |
| Section label | `st.subheader("Model CSV")` / `st.subheader("Observation CSV")` | Column section headers within Step 0 |
| Helper text | `st.caption(...)` | Secondary guidance below upload widgets |
| Instruction prose | `st.markdown(...)` — single sentence only | Top of step, below title |

**Do not use** `st.header()` anywhere in Phase 1 — it creates an inconsistent hierarchy with the existing step structure.

---

## Color

Streamlit controls all color. The contract below maps semantic feedback states to the correct Streamlit alert widgets. Using the wrong widget for a state breaks Streamlit's built-in color convention.

| State | Widget | When to Use |
|-------|--------|-------------|
| Success / valid input | `st.success(...)` | CSV uploaded and readable; columns detected |
| Warning / recoverable | `st.warning(...)` | CSV readable but something looks unusual (e.g., no obvious date column) |
| Error / blocking | `st.error(...)` | CSV cannot be parsed; required column not found |
| Neutral info | `st.info(...)` | Contextual guidance that is not an error (e.g., auto-detect note) |
| Destructive action | not present in Phase 1 | — |

**Accent (primary button):** `st.button(..., type="primary")` is reserved for the single forward-progress action per step. Only one primary button per step view. All other buttons use the default (secondary) style.

## Interaction Contract: Step 0 Changes

Phase 1 modifies Step 0 (`render_step_upload()`) only. The interaction contract for what changes:

### Upload Zone (columns 1 and 2)

**Model CSV column (col1):**
1. `st.subheader("Model CSV")` — unchanged heading
2. Remove `st.caption("Required columns: \`SimDate\`, \`Flow\`, \`TN\`, \`TP\`")` — replace with `st.caption("Upload any CSV with a date column and a numeric value column.")`
3. `st.file_uploader(...)` — unchanged widget
4. On upload: read headers with `pipeline_bridge.get_csv_columns()`. Do NOT validate for specific column names.
5. Show `st.success(f"{len(cols)} columns detected")` — no error for column mismatch.
6. Immediately below success: show `st.dataframe(preview_df, hide_index=False)` — 5 rows, read-only. Use `st.dataframe` not `st.table` (reason: `st.dataframe` renders faster and is horizontally scrollable for wide CSVs).
7. Below preview: show two dropdowns (stacked, not side-by-side):
   - `st.selectbox("Date column", cols, index=_best_guess_index(cols, ["date", "time", "datetime", "simdate", "timestamp"]))` — label: "Date column"
   - `st.selectbox("Value column", cols, index=_best_guess_index(cols, ["value", "flow", "discharge", "streamflow"]))` — label: "Value column (variable to reanalyse)"

**Observation CSV column (col2):**
- No structural changes. Caption, uploader, and success message remain as-is.
- Add 5-row preview below the existing `st.success(...)` using `st.dataframe(obs_preview_df)`.
- The observation column dropdowns remain in Step 1 — do not move them.

### Variable Selectbox — Remove Entirely

Remove the `st.selectbox("Variable to reanalyze", ["discharge", "TN", "TP"], ...)` widget from Step 0. The variable name is now the value column name the user selected from the model CSV dropdown. No replacement widget is needed.

### Station Name Field — Unchanged

`st.text_input("Station name (used as label in outputs)", ...)` — keep as-is, no change.

### Proceed Gate

`can_proceed` condition:
- `st.session_state.model_file is not None` — unchanged
- `st.session_state.obs_file is not None` — unchanged
- `st.session_state.model_date_col` is set (not empty) — new condition
- `st.session_state.model_value_col` is set (not empty) — new condition
- `station_name.strip() != ""` — unchanged

Primary CTA button: `st.button("Next: Configure Observation CSV →", disabled=not can_proceed, type="primary")` — label unchanged.

## Session State Contract

New keys added to `st.session_state` in Phase 1:

| Key | Type | Set When | Used By |
|-----|------|----------|---------|
| `model_date_col` | `str` | User selects from date dropdown | `_start_job()` → `build_model_df_generic()` |
| `model_value_col` | `str` | User selects from value dropdown | `_start_job()` → `build_model_df_generic()`, variable label throughout |

The existing `variable` key in session state is replaced by `model_value_col`. The string stored in `model_value_col` is the raw CSV column name (e.g., `"streamflow_cms"`). It is used as-is for output filenames, log messages, and UI labels per decision D-04.
